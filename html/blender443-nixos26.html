NixOS 26.05 // nix develop memo

Blender 4.4.3 を動かす

buildFHSEnv + nix develop — システムの blender5 はそのまま

概要

NixOS 26.05 のデフォルト pkgs.blender は 4.5.x。公式バイナリの 4.4.3 を buildFHSEnv で包んで、nix develop 中だけ使えるようにする。

Note: blender-4.4.3-linux-x64.tar.xz は blender.org から手動DLして展開しておく。flake.nix と同じディレクトリに置く。

ディレクトリ構成

tree
~/flakes/blender4/
├── blender-4.4.3-linux-x64/   ← 展開済みバイナリ
│   └── blender
├── flake.nix
├── flake.lock
└── .gitignore

flake.nix

flake.nix
{
  description = "Blender 4.4.3 FHS env";

  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-26.05";

  outputs = { self, nixpkgs }:
  let
    system = "x86_64-linux";
    pkgs   = import nixpkgs { inherit system; };

    blenderFHS = pkgs.buildFHSEnv {
      name = "blender-fhs";

      targetPkgs = pkgs: with pkgs; [
        libx11
        libxext
        libxrender
        libxfixes
        libxi
        libxrandr
        libxxf86vm
        libxkbcommon
        libSM
        libICE
        libxcb
        libXdmcp
        libXau
        dbus
        libGL
        libGLU
        glib
        fontconfig
        freetype
        zlib
        alsa-lib
        pipewire
        wayland
        vulkan-loader
      ];

      runScript = pkgs.writeShellScript "run-blender" ''
        exec "$BLENDER_DIR/blender" "$@"
      '';
    };
  in {
    devShells.${system}.default = pkgs.mkShell {
      packages = [ blenderFHS ];

      shellHook = ''
        export BLENDER_DIR="$PWD/blender-4.4.3-linux-x64"
        echo "blender-fhs で Blender 4.4.3 が起動します"
      '';
    };
  };
}

使い方

1
blender4 ディレクトリに移動
cd ~/flakes/blender4
2
FHS 環境に入る
nix develop
3
Blender 4.4.3 を起動
blender-fhs
4
終了は exit または Ctrl-D — システムの blender5 に戻る

不足ライブラリが出た場合

エラー例: error while loading shared libraries: libFoo.so.1

nix-locate で調べる
nix-locate --top-level libFoo.so.1

出てきたパッケージ名を targetPkgs リストに追記して nix develop し直す。
