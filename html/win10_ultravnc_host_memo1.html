Windows 10 VNCサーバー構築 備忘録

目的: Linux環境（Remmina）からWindows 10をリモートデスクトップ操作する（再起動後・サインイン前でも自動待ち受け）
最終採用構成: UltraVNC (Server) + Windowsシステムサービス化 (Port: 5900)

1. トラブルシューティングの軌跡（不採用ルート）

  • TigerVNC (GUIインストーラー): 通常のNext連打ではViewerしか入らず、Server機能（winvnc.exe）がインクルードされない罠がある。
  • TigerVNC (スタンドアロン配置): SourceForgeからの直接ダウンロード時にファイルが破損（Corrupted / Unreadable）し、起動失敗。
  • TigerVNC (GitHub ZIPルート): 公式リリースのURL構造変更、あるいは名称不一致により 404 Not Found でダウンロード不能。
教訓: GUIインストーラーのバグや、海外ミラーのプロキシ起因の破損に依存しない「パッケージマネージャー（winget）による代替ツールの導入」へ舵を切ったのがブレイクスルーとなった。

2. 【確定版】UltraVNC 構築コマンド（管理者権限のPowerShell）

以下は、今回開通に至った全手順のコマンドです。次回再構築する際はこのブロックを順番に実行するだけで完了します。

ステップ①: 本体インストール

# wingetを使ってUltraVNC（サーバー入りフルパッケージ）をサイレントインストール
winget install uvncbvba.UltraVnc --accept-source-agreements --accept-package-agreements

ステップ②: パスワードの注入

# パスワードを変数に代入し、設定用EXEへ引数として渡す
$pass = "設定したパスワード"
& "C:\Program Files\uvnc bvba\UltraVNC\Setpasswd.exe" $pass

ステップ③: サーバー強制起動と常駐・自動起動設定（Restart対策）

# 既存プロセスを一度終了
Stop-Process -Name "winvnc" -Force -ErrorAction SilentlyContinue

# UltraVNCを「Windowsシステムサービス」としてシステムに正式登録
& "C:\Program Files\uvnc bvba\UltraVNC\winvnc.exe" -install

# サービス名を自動検出して、スタートアップの種類を「自動（常駐）」に変更＆サービス開始
$vncService = Get-Service | Where-Object {$_.Name -like "*uvnc*" -or $_.DisplayName -like "*uvnc*"}
if ($vncService) {
    Set-Service -Name $vncService.Name -StartupType Automatic
    Start-Service -Name $vncService.Name
}

3. 状態確認用コマンド

Windows側での待ち受け（LISTEN）確認

netstat -ano | Select-String "5900"

成功時の出力例: TCP 0.0.0.0:5900 ... LISTENING が出ればWindows側は正常。

サービスの自動起動ステータス確認

Get-Service | Where-Object {$_.Name -like "*uvnc*"} | Select-Object Name, Status, StartType

成功時の出力例: Status: Running / StartType: Automatic

4. Linux（Remmina）からの接続方法

  • プロトコル: VNC - Virtual Network Computing
  • サーバーアドレス: [WindowsのIPアドレス]:5900 （例: 192.168.1.XX:5900）
  • パスワード: 上記ステップ②で指定したパスワード

※Windows側のファイアウォール（ポート5900）は以前の工程で開放済みのため、これでいつでもホスト名やIP指定だけで即座にデスクトップに繋がります。
