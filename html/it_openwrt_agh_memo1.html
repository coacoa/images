OpenWrt AdGuard Home 設定メモ

機種: I-O DATA WN-AX2033GR (OpenWrt 23.05.6)

重要: /etc/adguardhome/ 配下に設定を置くことで、再起動しても設定が消えないように構築済み。

1. AdGuard Home のインストール

opkg update
opkg install adguardhome
mkdir -p /etc/adguardhome
/etc/init.d/adguardhome enable
/etc/init.d/adguardhome start

2. DNS 連携設定 (dnsmasq & AGH)

標準の 53番ポートを dnsmasq が受け、5353番の AGH へ投げる構成。

# dnsmasq の問い合わせ先を AGH に固定
uci del dhcp.@dnsmasq[0].server
uci add_list dhcp.@dnsmasq[0].server='127.0.0.1#5353'
uci set dhcp.@dnsmasq[0].noresolv='1'
uci set dhcp.@dnsmasq[0].localuse='1'

# クライアントに自分(192.168.250.1)を DNS として教える
uci del dhcp.lan.dhcp_option
uci add_list dhcp.lan.dhcp_option='6,192.168.250.1'

uci commit dhcp
/etc/init.d/dnsmasq restart

3. ネットワーク疎通 (NAT/マスカレード)

二重ルーター環境で配下のデバイスがネットに繋がるための必須設定。

uci set firewall.@zone[1].masq='1'
uci set firewall.@zone[1].mtu_fix='1'
uci commit firewall
/etc/init.d/firewall restart

4. WAN 側からのアクセス許可

192.168.1.x (上流ネットワーク) から管理画面を開くための穴あけ。

# LuCI (80ポート) と AGH (8080ポート) を許可
uci add firewall rule
uci set firewall.@rule[-1].name='Allow-WAN-Admin'
uci set firewall.@rule[-1].src='wan'
uci set firewall.@rule[-1].dest_port='80 8080'
uci set firewall.@rule[-1].proto='tcp'
uci set firewall.@rule[-1].target='ACCEPT'
uci commit firewall
/etc/init.d/firewall restart

5. 動作確認コマンド

  • AGH 直接テスト: drill @127.0.0.1 -p 5353 google.com
  • dnsmasq 経由テスト: drill @127.0.0.1 google.com
