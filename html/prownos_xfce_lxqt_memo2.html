x11vnc 完全安定版（PrawnOS + SDDM + LXQt）

このメモは、PrawnOS（Debian Bullseye）で発生する Remmina の read 104 問題を完全に解決するための 最終安定構成をまとめたもの。

■ 問題の正体

PrawnOS + SDDM + LXQt 環境では、Xorg が以下のように動作する：

  • SDDM と LXQt が同じ Xorg プロセスを共有する
  • auth は /var/run/sddm/{UUID} に生成される
  • UUID は再起動ごとに変わる
  • x11vnc が古い UUID を参照すると即終了 → read 104

つまり、read 104 の原因は auth mismatch であり、 UUID を毎回自動検出すれば完全に解決する。

■ SDDM autologin（必須）

以下のファイルを作成：

sudo nano /etc/sddm.conf.d/autologin.conf
内容： 
[Autologin]
User=hoge
Session=lxqt.desktop
Relogin=true

■ x11vnc を “UUID 自動検出 + DISPLAY=:0 固定” にする（最終形）

以下の systemd ユニットを作成：

sudo nano /etc/systemd/system/x11vnc.service
内容： 
[Unit]
Description=Start x11vnc after LXQt session
After=graphical-session.target

[Service]
Type=simple

# 起動時に /var/run/sddm の UUID を自動検出
ExecStartPre=/bin/sh -c 'sleep 2; AUTH=$(ls /var/run/sddm/* | head -n 1); echo $AUTH > /tmp/x11vnc_auth'

# LXQt の Xorg (:0) に接続
ExecStart=/bin/sh -c '/usr/bin/x11vnc -auth $(cat /tmp/x11vnc_auth) \
 -noxdamage -usepw -rfbauth /etc/x11vnc.pass \
 -forever -loop -shared -repeat -display :0'

[Install]
WantedBy=graphical-session.target
反映： 
sudo systemctl daemon-reload
sudo systemctl enable x11vnc
sudo systemctl restart x11vnc

■ これで安定する理由

  • UUID 自動検出 → 再起動しても auth mismatch が起きない
  • DISPLAY=:0 固定 → SDDM の Xorg を掴まない
  • graphical-session.target → LXQt 起動後に x11vnc を開始
  • sleep 2 → autologin 完了まで待つ

これにより、x11vnc は常に正しい Xorg と auth を掴み、 read 104 は永久に発生しなくなる。

■ 動作確認

sudo systemctl status x11vnc
journalctl -u x11vnc -n 50

Remmina から vnc://IP:5900 で接続。

■ 最終まとめ

  1. SDDM autologin を /etc/sddm.conf.d/ に設定
  2. Xorg の auth は /var/run/sddm/UUID
  3. UUID は毎回変わる → 自動検出が必須
  4. x11vnc を UUID 自動検出 + DISPLAY=:0 にする
  5. これで read 104 は完全に消える

この構成は PrawnOS + SDDM + LXQt 環境で最も安定する “最終解決版”。
