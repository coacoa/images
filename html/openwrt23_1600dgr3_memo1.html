OpenWrt DNS & 安定化マニュアル

1. DNS連携設定 (Dnsmasq ⇄ AGH)

ポート53のDnsmasqから、ポート5353のAGHへクエリを転送し、AGH停止時のバックアップ(8.8.8.8)も設定済み。

# AGHをメイン、Googleを予備に設定
uci del dhcp.@dnsmasq[0].server
uci add_list dhcp.@dnsmasq[0].server='127.0.0.1#5353'
uci add_list dhcp.@dnsmasq[0].server='8.8.8.8'

# 上位ルーターからの不安定なDNSを無視
uci set dhcp.@dnsmasq[0].noresolv='0'

uci commit dhcp
/etc/init.d/dnsmasq restart

2. メモリ不足・フリーズ対策 (Swap設定)

128MB RAMの限界をExtroot上のSwapファイル(512MB)で補強。LED全点灯フリーズを防止。

# Swapファイルの作成と有効化
dd if=/dev/zero of=/overlay/swap.file bs=1M count=512
chmod 600 /overlay/swap.file
mkswap /overlay/swap.file
swapon /overlay/swap.file

# 自動マウント設定
uci add fstab swap
uci set fstab.@swap[-1].device='/overlay/swap.file'
uci set fstab.@swap[-1].enabled='1'
uci commit fstab
確認コマンド: free -m を叩き、Swapのtotalが512MB付近なら正常。

3. トラブルシューティング

症状 確認・対策
ネットが重い free -m でSwapの使用量を確認。多すぎるなら reboot。
特定の端末だけ「接続なし」 端末側の「プライベートDNS」がオフになっているか確認。
AGH管理画面が開かない /etc/init.d/adguardhome restart でAGHを叩き起こす。

4. 構成概要図

端末 → [Port 53] Dnsmasq → [Port 5353] AdGuard Home (フィルタ17万行)
※メモリが溢れたら /overlay/swap.file (512MB) へ一時退避
