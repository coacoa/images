OpenWrt AdGuard Home 完全復旧マニュアル

WRC-1167GST2 (OpenWrt 23.05) で AdGuard Home を安定稼働させるための全手順です。

基本情報:
ルーターIP: 192.168.253.1
AGH管理画面: http://192.168.253.1:8080
DNS構成: PC(53) → dnsmasq(53) → AGH(5353)

STEP 1: パッケージのインストール

opkg update opkg install adguardhome /etc/init.d/adguardhome enable /etc/init.d/adguardhome start

STEP 2: AdGuard Home の初期設定 (GUI)

ブラウザで http://192.168.253.1:3000 を開き、ウィザードを進める。

  • 管理画面ポート: 8080
  • DNSサーバーポート: 5353

STEP 3: ポート競合の解消とバケツリレー設定

SSHで以下のコマンドを順に実行。dnsmasqを53番の窓口にし、中身を5353番のAGHに丸投げする設定です。

# dnsmasqを53番に戻し、リレー先をAGH(5353)にする uci set dhcp.@dnsmasq[0].port='53' uci set dhcp.@dnsmasq[0].noresolv='1' uci del_list dhcp.@dnsmasq[0].server uci add_list dhcp.@dnsmasq[0].server='127.0.0.1#5353' # クライアントに自分をDNSとして教える uci del dhcp.lan.dns uci add_list dhcp.lan.dns='192.168.253.1' uci commit dhcp /etc/init.d/dnsmasq restart

STEP 4: ファイアウォールの開放

通信が止まらないよう、LAN側からの入力を確実に許可します。

uci set firewall.@zone[0].input='ACCEPT' uci set firewall.@zone[0].forward='ACCEPT' uci set firewall.@zone[1].masq='1' uci commit firewall /etc/init.d/firewall restart

STEP 5: 設定の永久保存 (重要)

電源を切っても消えないように、キャッシュを物理メモリに書き込みます。

uci commit sync # これで電源を抜いても安心！
困ったときは？
もしネットが死んだら、PCのターミナルで ping 8.8.8.8 を打つ。
通るならDNSの問題（AGH停止など）、通らないならWANの問題（ケーブルやF660Tとの接続）です。
