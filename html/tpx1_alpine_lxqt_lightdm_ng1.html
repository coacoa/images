```html Alpine 3.23 × X1 Carbon 3rd Gen GPU 悲劇メモ

Alpine 3.23 GPU 非対応による Xorg 崩壊の悲劇メモ

概要

ThinkPad X1 Carbon 3rd Gen（Ivy Bridge / Intel HD 4000）で Alpine Linux 3.23 を使った結果、 Mesa から i965 ドライバが消えており、Xorg が起動不能になった。 その巻き添えで LightDM の autologin も x11vnc も全部死んだ、という悲劇の記録。

症状

  • LightDM が起動しない or すぐ TTY に落ちる
  • autologin が一切発動しない
  • x11vnc が接続できない（DISPLAY=:0 が存在しない）
  • 画面は常に alpine login: の TTY のまま

Xorg のログには毎回これが出る：

(EE) AIGLX error: dlopen of /usr/lib/dri/i965_dri.so failed (Error loading shared library /usr/lib/dri/i965_dri.so: No such file or directory)
(EE) AIGLX error: unable to load driver i965

調査ログ

Xorg のエラー

cat /var/log/Xorg.0.log | grep EE

(EE) Failed to load module "fbdev" (module does not exist, 0)
(EE) Failed to load module "vesa" (module does not exist, 0)
(EE) AIGLX error: dlopen of /usr/lib/dri/i965_dri.so failed
(EE) AIGLX error: unable to load driver i965

/usr/lib/dri の中身

ls -l /usr/lib/dri/

lrwxrwxrwx 1 root root    14 May  8 21:28 crocus_dri.so -> libdril_dri.so
lrwxrwxrwx 1 root root    14 May  8 21:28 i915_dri.so   -> libdril_dri.so
-rwxr-xr-x 1 root root 8011848 May 10  2025 i965_drv_video.so
lrwxrwxrwx 1 root root    14 May  8 21:28 iris_dri.so   -> libdril_dri.so
...

決定的ポイント： i965_dri.so が存在しない。
あるのは i915 / crocus / iris などだけ。 Ivy Bridge (HD 4000) が本来使うべき i965 が完全に消えている。

なぜ直らないのか

1. 「mesa-dri」がそもそも存在しない

apk add mesa-dri
ERROR: unable to select packages:
  mesa-dri (no such package)

Alpine 3.23 には mesa-dri パッケージ自体が存在しない。 community リポジトリを有効化しても同じ。

2. Mesa の構成変更で i965 が削除されている

  • 古い Intel GPU 用の i965_dri.so がパッケージから削除
  • 代わりに crocus / iris / Gallium 系に移行
  • しかし Ivy Bridge (HD 4000) はそれらの対象外

結果として：

  • Xorg は i965 をロードしようとして失敗
  • GPU 初期化に失敗 → Xorg 自体が落ちる
  • LightDM は Xorg を起動できないので autologin も始まらない
  • x11vnc は DISPLAY=:0 が無いので接続できない

これは設定ミスではなく「Alpine 3.23 の Mesa が Ivy Bridge を切り捨てた」という仕様の問題。

やっても無駄だったこと一覧

  • mesa, mesa-gl, mesa-egl, mesa-gbm のインストール・再インストール
  • xf86-video-intel の導入
  • LightDM の greeter-session / autologin-user / autologin-session 調整
  • SDDM の削除、LightDM 単独運用
  • community リポジトリ有効化 → apk update

どれだけ頑張っても、存在しない i965_dri.so は生えてこないので根本解決にはならない。

最終結論

Alpine 3.23 は Ivy Bridge (Intel HD 4000) を Mesa レベルでサポートしていない。
そのため X1 Carbon 3rd Gen では：

  • Xorg → 起動不能
  • LightDM → Xorg が死ぬので autologin 不可
  • x11vnc → 共有する画面が無いので接続不可

つまり、どれだけ設定をいじっても「3.23 では GUI は動かない」という結論になる。

唯一の解決策

Alpine のバージョンを下げる

Alpine 3.20（または 3.18 / 3.19）に戻す。
これらのバージョンではまだ i965_dri.so が含まれており、以下が正常に動く：

  • Xorg
  • LightDM + autologin
  • LXQt
  • x11vnc（Remmina からのリモート GUI）

再構築のざっくり手順

  1. Alpine 3.20 の ISO をダウンロード
  2. USB に書き込む（Ventoy でも dd でも可）
  3. sys モードでインストールし直す
  4. LXQt + LightDM + x11vnc を再セットアップ
  5. autologin & リモートデスクトップ環境が復活

一言メモ

設定が悪いんじゃなくて、時代の流れで i965 が切り捨てられた側だった、というオチ。
「直せないバグ」じゃなくて「そのバージョンではそもそも動かない仕様」だと分かっただけでも収穫。
```
