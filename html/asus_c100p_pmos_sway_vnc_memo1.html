01

eMMCへの書き込みトラブル

⚠ 注意
C100PのeMMCに書き込む際、buffer I/O error や lost async page write が発生することがある。

原因の調査コマンド：

sudo dmesg | grep "mmcblk0" | tail -20
sudo cat /sys/block/mmcblk0/ro         # 0なら書き込み可能
cat /sys/block/mmcblk0boot0/force_ro  # 1なら保護されている

boot保護を解除：

echo 0 | sudo tee /sys/block/mmcblk0boot0/force_ro
echo 0 | sudo tee /sys/block/mmcblk0boot1/force_ro
⚠ NG例
seek=2048 をつけるとブートローダーが飛んでChromeOS recoveryになる。絶対につけない。
02

正しいddコマンド

使用イメージ
google-veyron (RK3288) 用イメージを使うこと。C101PA用の google-gru (RK3399) は不可。

img.xzを展開しながら直接書き込む（推奨）：

xzcat /home/user/postmarketos-veyron.img.xz | sudo dd of=/dev/mmcblk0 bs=1M oflag=direct iflag=fullblock
✓ ポイント
  • oflag=direct：キャッシュをバイパス
  • iflag=fullblock：完全ブロック読み込み
  • status=progress：PostmarketOSのddでは非対応のため省略
  • img.xzをそのままxzcatに渡す（展開不要）

lsblkでデバイス確認：

lsblk -o NAME,SIZE,TYPE,MOUNTPOINT
デバイス説明
mmcblk0 (14.7GB)内蔵eMMC（書き込み先）
mmcblk0boot0/1 (4MB)ブート用特殊パーティション
sdaUSBメモリ（イメージ元）
mtdblock0 (4MB)ファームウェア用フラッシュ（触らない）
03

VNC / RDPリモート接続

GNOMEはWayland上で動作するため、x11vncやxrdpとの相性が悪い。Swayに変更してwayvncを使うのが最も確実。

ファイアウォールの無効化（接続できない場合）：

sudo nft flush ruleset
⚠ xrdpの問題点
  • 証明書がない → opensslで生成が必要
  • sesmanがXorg起動に失敗（WaylandとVT競合）
  • GNOME+Mutterはwlr-screencopyを実装していない

wayvncのインストールと起動（Sway環境）：

sudo apk add wayvnc
export WAYLAND_DISPLAY=wayland-1
export XDG_RUNTIME_DIR=/run/user/$(id -u)
wayvnc 0.0.0.0
✓ Remmina VNC接続設定

04

Sway セットアップ

sudo apk add sway wayvnc foot wofi

設定ファイルのコピー：

mkdir -p ~/.config/sway
cp /etc/sway/config ~/.config/sway/config

wofiランチャーの追加：

echo "bindsym Mod4+d exec wofi --show run" >> ~/.config/sway/config

SSH自動環境変数（~/.profileに追記）：

export WAYLAND_DISPLAY=wayland-1
export XDG_RUNTIME_DIR=/run/user/$(id -u)
export SWAYSOCK=$(ls /run/user/$(id -u)/sway-ipc.*.sock 2>/dev/null | head -1)
05

SSHから操作する方法

swaymsgのsocketを確認：

ls /run/user/$(id -u)/sway-ipc.*.sock

アプリをSSHから起動：

WAYLAND_DISPLAY=wayland-1 \
XDG_RUNTIME_DIR=/run/user/$(id -u) \
SWAYSOCK=/run/user/$(id -u)/sway-ipc.*.sock \
swaymsg exec foot

Swayをリロード：

SWAYSOCK=/run/user/$(id -u)/sway-ipc.*.sock swaymsg reload
⚠ 注意
再起動するたびにsocketのPID番号が変わる。~/.profileに自動取得の設定を入れておくと便利。
