監視カメラ運用メモ

WXR-2533DHP (OpenWrt 24.10) + Logicool C615 + NixOS miniPC

電源ON後の起動フロー

電源ON
USBメモリ（extroot）が自動マウントされる
約30〜60秒 — OpenWrt 起動完了
kmod-video-uvc が読み込まれ /dev/video0 が出現
mjpg-streamer 自動起動 (START=99)
640×480 / 15fps でストリーミング開始
miniPC (NixOS) の motion が自動再接続
ストリームURLを検知 → 動体検知・録画再開

ストリームURL

mjpg-streamer 稼働中
解像度640 × 480
フレームレート15 fps
ポート8080

ライブ映像（ブラウザで直接開ける）

http://192.168.1.1:8080/?action=stream

静止画スナップショット

http://192.168.1.1:8080/?action=snapshot

電源ON/OFF 運用

安全
電源を切っても extroot・設定・自動起動はすべて保持されます。必要なときだけONにする運用は完全に問題ありません。
注意
電源を切る前に特別な操作は不要ですが、パッケージインストール中・設定保存中は避けてください。
電源OFF
extroot保持
fstab設定保持
自動起動設定保持
電源ON後
USB自動マウント自動
カメラ映像自動
motion再接続自動

miniPC (NixOS) — motion 設定

motion.conf — ストリーム受信設定NixOS側
# ストリームソース
netcam_url      http://192.168.1.1:8080/?action=stream
netcam_keepalive on

# 動体検知設定
threshold       1500    # 感度（小さいほど敏感）
minimum_motion_frames 2

# 録画保存先
target_dir      /var/lib/motion
ffmpeg_output_movies on
movie_filename  %Y%m%d-%H%M%S

# ストリーム切断時に自動再接続
netcam_tolerant_check on
ポイント
netcam_tolerant_check を on にするとルーターの電源OFF/ON時にmotionが自動で再接続を試みます。
NixOS — configuration.nixNixOS側
services.motion = {
  enable = true;
  settings = {
    netcam_url = "http://192.168.1.1:8080/?action=stream";
    netcam_keepalive = "on";
    netcam_tolerant_check = "on";
    target_dir = "/var/lib/motion";
    ffmpeg_output_movies = "on";
    threshold = "1500";
  };
};

トラブルシュート

映像が出ない場合確認手順
# mjpg-streamerの状態確認
ps | grep mjpg

# カメラデバイス確認
ls /dev/video*

# 手動で再起動
/etc/init.d/mjpg-streamer restart

# ログ確認
logread | grep mjpg
