AS3 と HTML / CSS の object・instance・class 対応メモ

結論まとめ

  • CSS の .class は AS3 の class（設計図）ではない
  • HTML 要素（DOM要素）そのものが instance
  • CSS は「個別操作」ではなく「条件一致した集合」を操作する

AS3 の考え方（おさらい）

class Box {
  public var color:String;
}

var box1:Box = new Box();
var box2:Box = new Box();

box1.color = "red";
box2.color = "blue";
  • class = 設計図
  • instance = 実体
  • box1.color = 個別インスタンス操作

HTML / CSS の考え方

<div class="box"></div>
<div class="box"></div>
.box {
  color: red;
}
.box は「boxというラベルが付いた要素すべて」を意味する。

対応表

AS3HTML / CSS
classCSS class（意味は別物）
instanceDOM要素
instance名基本的に存在しない
box1.colorJS: element.style.color

HTML での「インスタンス名」に近いもの

<div id="box1" class="box"></div>
#box1 {
  color: red;
}
document.getElementById("box1");

id は AS3 のインスタンス名に一番近い概念。

class は何に近いのか？

AS3 で複数インスタンスに同じ属性を付けるための「タグ」「属性」に近い。 

// AS3 的発想
box1.type = "enemy";
box2.type = "enemy";
/* CSS */
.enemy { }

決定的な違い

  • AS3：命令型（1個ずつ操作）
  • CSS：宣言型（条件に一致する集合を操作）
.box.highlight button:hover {
  background: red;
}

JS を使うと AS3 感覚に近づく

const box1 = document.getElementById("box1");
box1.style.color = "red";

直感的なたとえ

  • AS3：「box1 に赤を塗れ」
  • CSS：「条件を満たす箱は赤」

まとめ

  • CSS class は設計図ではない
  • DOM要素 = インスタンス
  • id = インスタンス名に最も近い
  • Flash経験者は JS と CSS を役割分担で理解すると楽
