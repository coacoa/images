Upscale Project: 振り返りと次戦への備え

【教訓】 ncnn-vulkan版はNixOS環境下でのドライバ/VRAM制御が極めて不安定。 メモリリソースを過信せず、より軽量または安定したバックエンドへの切り替えを推奨。

🚀 次回試すべき「別の武器」

1. Waifu2x-deadconv (Caffe/OpenCL版)

Real-ESRGANより軽量で、古いGPUやOpenCL環境でも安定して動くことが多いです。実写動画でも「デノイズ」目的で非常に優秀です。

  • 利点: メモリ消費が少なく、壊れた画像を吐きにくい。
  • NixOS: waifu2x-converter-cpp などのパッケージを確認。

2. FFmpeg 内蔵フィルター (Super Resolution)

外部ツールを使わず、FFmpegの中でAIモデル（.pbファイルなど）を読み込んで直接アップスケールします。

  • 利点: フレーム分解・再結合の手間がなく、今回のような引数エラーが起きない。
  • 方法: -vf sr=model=path/to/model.pb を使用。

3. Python + PyTorch (onnxruntime)

ncnn（C++）のバイナリではなく、Python環境でONNXモデルを動かします。

  • 利点: CPU/GPUの切り替えが柔軟で、エラーメッセージが詳細。
  • NixOS: mach-nix や poetry2nix で環境を固定して実行。

🛠 根本的な解決へのチェックリスト

  • VRAM管理: タイルサイズを 128 以下にするか、スケーリング倍率を -s 2 に固定する。
  • ドライバの確認: nvidia-smi でGPUが認識されているか、vulkaninfo でVulkan層が生きているか確認。
  • 一時ファイルの排除: 実行前に必ず rm -rf /tmp/upscale_* 等で残骸を消す。

※ 今回の `upscale.sh` のロジック（FFmpegの結合など）は、他のツールに切り替えてもそのまま応用可能です。
