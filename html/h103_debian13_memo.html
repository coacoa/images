ASUS H103 / LXQt 再セットアップ用メモ

環境前提：

  • ASUS H103（Atom x5）
  • Debian + LXQt + SDDM
  • キーボード物理故障 → 自動ログイン前提
  • 画面は DSI-1、初期状態は縦（800x1280）

1. SDDM で自動ログイン + LXQt を有効化

/etc/sddm.conf を作成または編集：

[Autologin]
User=hoge
Session=lxqt.desktop

[General]
HaltCommand=/usr/bin/systemctl poweroff
RebootCommand=/usr/bin/systemctl reboot

保存後、再起動：

sudo reboot

2. 画面回転の確認（SSH から）

SSH でログインして、現在のディスプレイ状態を確認：

DISPLAY=:0 xrandr

例：

Screen 0: minimum 320 x 200, current 800 x 1280, ...
DSI-1 connected primary 800x1280+0+0 ...

横向きに回転：

DISPLAY=:0 xrandr -o right

再確認：

DISPLAY=:0 xrandr
# current 1280 x 800 になっていれば OK

3. タッチデバイスの確認

SSH から：

DISPLAY=:0 xinput list

重要なデバイス：

ELAN2562:00 04F3:2562

4. タッチ座標の補正（right 回転時）

画面を -o right にした状態で：

DISPLAY=:0 xinput set-prop "ELAN2562:00 04F3:2562" --type=float "Coordinate Transformation Matrix" 0 1 0 -1 0 1 0 0 1

5. LXQt 起動時に「遅延つき」で自動回転 + タッチ補正

autostart ディレクトリを作成：

mkdir -p ~/.config/autostart

~/.config/autostart/rotate.desktop を作成：

[Desktop Entry]
Type=Application
Exec=sh -c "sleep 4 && xrandr -o right && xinput set-prop 'ELAN2562:00 04F3:2562' --type=float 'Coordinate Transformation Matrix' 0 1 0 -1 0 1 0 0 1"
Hidden=false
NoDisplay=false
X-GNOME-Autostart-enabled=true
Name=RotateFix

ポイント：

  • sleep 4 が重要（DSI-1 初期化完了後に回転させるため）
  • -o right は、この個体で横向きになる向き
  • タッチ補正も同時に適用

設定後、再起動：

sudo reboot

期待動作：

  • 起動直後：縦（800x1280）
  • 数秒後（約 4 秒）：自動で横（1280x800）に回転
  • タッチ座標も正しく一致

6. トラブル時の復旧メモ

画面が真っ黒になった場合（ブラックアウト）

SSH でログインして autostart を無効化：

mv ~/.config/autostart/rotate.desktop ~/.config/autostart/rotate.desktop.disabled
sudo reboot

これで「縦画面の LXQt 起動」に戻る。
