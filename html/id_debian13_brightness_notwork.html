ASUS T103HAF 明るさ設定：格闘の記録

現状の課題: スライダーを動かしても本体の明るさが変わらない。
原因: カーネルが acpi_video0~7 というダミーの制御ファイルを8個も作ってしまい、本物の制御 (intel_backlight) が無視されている。

1. これまでの試行結果

  • acpi_backlight=none: ダミーは消えたが、本物も消えてスライダーが消失。
  • acpi_backlight=native: 危険。 画面が完全にブラックアウト（消灯）したため中止。
  • 現状: デフォルト設定に戻し、眩しさは「夜間モード（Night Light）」で軽減中。

2. 今後の解決案 A：ソフトウェア制御（推奨）

ハードウェアのバックライト（物理的なLED）をいじるのが危険なため、画面に映る「ピクセルの色」を計算で暗くする方法です。これなら絶対にブラックアウトしません。

sudo apt update
sudo apt install brightness-controller

使い方: メニューから起動し、スライダーを下げるだけ。OSのスライダーが効かなくても、これで確実に暗くできます。

3. 今後の解決案 B：コマンドによる強制叩き

もし物理的にバックライトを操作したい場合、OSのスライダーを経由せずに直接数値を送り込みます。

# brightnessctl をインストール
sudo apt install brightnessctl

# 現在の明るさを確認
brightnessctl info

# 50% に設定（テスト済み。もし真っ暗になったら SSH から 100% に戻せる）
sudo brightnessctl set 50%

4. 今後の解決案 C：カーネルパラメータの微調整

次に試す価値がある（真っ暗になりにくい）パラメータ：

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash pcie_aspm=off"

※電力管理周りの相性問題でバックライトが固定される場合があるため。

結論: まずは brightness-controller を使って「ソフト的に暗くする」のが、ASUS T103HAF においては最も安全で確実な回避策です。
