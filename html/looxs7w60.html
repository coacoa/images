LOOX × WinME × DSL 起動チャレンジ：戦いの全記録

このメモは、WinME が搭載された LOOX を DSL（Damn Small Linux）で起動しようとした際の、全ての試行錯誤をまとめたもの。

1. 最初の方針：WinME の DOS から grub4dos を起動する

当初の作戦は、以下の流れを実現することだった：

WinME → DOS → autoexec.bat → C:\grldr → grub4dos → DSL

しかし、WinME には「Real DOS Mode を封印する」仕様があり、これが最大の障害となった。

2. msdos.sys の編集と invalid エラー

msdos.sys を編集して以下を設定：

[Options]
BootGUI=0
BootMulti=1
SuppressBootMenu=0

しかし起動時に一瞬だけ

invalid setting in msdos.sys file

と表示され、WinME がそのまま起動してしまう。

これは WinME の DOS 封印が原因で、書式ミスではなかった。

3. Real DOS-Mode Patch を試すが、INF が動かない

WinME の DOS 封印を解除するため、GitHub の DOSMODE.INF を入手。

しかし INF を右クリック → インストールすると：

  • command.com が見つからない
  • exit_dos.pif が見つからない

などのエラーが発生し、インストールが完走しない。

これは WinME が DOS 関連ファイルを「隠す・封印する・アクセス拒否する」ため。

4. loadlin に切り替えるが “not enough memory”

WinME を経由せず Linux を起動するため、loadlin を使用：

loadlin linux24 initrd=minirt24.gz root=/dev/ram0 rw

しかし WinME 通常モードでは：

not enough memory

Safe Mode でも同じエラー。

理由は WinME が Safe Mode でもメモリを大量に消費し、loadlin が必要とする「連続した低メモリ領域」が確保できないため。

5. 結論：WinME 経由の Linux 起動は LOOX ではほぼ不可能

ここまでの結果から、以下が判明：

  • WinME の DOS 封印は非常に強力で、msdos.sys では解除できない
  • DOSMODE.INF も WinME 側の封印で動かない
  • loadlin は WinME が RAM を使いすぎて動かない
  • Safe Mode でもメモリ不足は解消されない

つまり、WinME を経由して DSL を起動する方法は、LOOX の 128MB RAM では物理的に不可能。

6. 今後の現実的な選択肢

WinME を完全に無視して Linux を起動する方法が必要。

候補：

  • F8 → Command prompt only → loadlin（最軽量）
  • WinME 起動ディスク（フロッピー）から DOS → loadlin
  • Win98 の IO.SYS / COMMAND.COM を移植して DOS を復活させる

いずれも WinME の DOS 封印を回避するための方法。

7. まとめ

今回の格闘でわかったこと：

  • WinME の DOS 封印は msdos.sys では解除できない
  • INF も封印のせいで動かない
  • loadlin は WinME のメモリ管理と相性最悪
  • 128MB RAM の LOOX では WinME 経由の Linux 起動は困難
  • WinME を経由しない DOS 起動が必要

ここまでの試行錯誤はすべて正しい方向だったが、WinME の仕様が壁になった。

次に進むなら、WinME を完全にバイパスする方法が最も現実的。
