Alpine Linux: LXQt + i3 構築メモ

SSH接続した状態から、一般ユーザー hoge に sudo 権限を付与し、フリーズ対策を施した上で LXQt + i3 環境を構築する手順のまとめです。

💡 コマンドのルール:
Alpine Linux のサービス管理コマンドは、アンダースコア（_）ではなくハイフン（-）を使用します（例: rc-update, rc-service）。

1. rootでの初期設定 (sudoの導入)

まずは su コマンドで root ユーザーに切り替えて実行します。

1 コミュニティリポジトリの有効化

vi /etc/apk/repositories

※ URLの末尾が /community で終わっている行の先頭にある # を削除して保存（:wq）します。

2 パッケージ更新と sudo のインストール

apk update
apk add sudo

3 wheel グループの権限解放

visudo

※ # %wheel ALL=(ALL:ALL) ALL の行を探し、先頭の # を削除して保存します。

4 hoge ユーザーをグループに追加

addgroup hoge wheel

設定が終わったら、root を抜けます。

exit

2. 操作不能（フリーズ）防止の対策

ここからは hoge ユーザーで sudo を使って実行します。

GUI起動時にキーボードやマウスが動かなくなるのを防ぐため、入力用ドライバーとデバイス管理ツールを事前に入れます。

sudo apk add eudev xf86-input-libinput xf86-input-evdev
sudo rc-update add udev sysinit

3. GUI と i3wm のインストール

LXQtデスクトップ、i3ウィンドウマネージャー、およびログイン画面（SDDM）をまとめて導入します。

sudo apk add lxqt-desktop lxqt-core lxqt-panel sddm i3wm i3status dmenu alacritty

4. LXQt の ウィンドウマネージャーを i3 に指定する

LXQtセッションが立ち上がるときに i3 を使うよう、設定ファイルを作成します。

mkdir -p ~/.config/lxqt
vi ~/.config/lxqt/session.conf

ファイルに以下の内容を書き込んで保存します：

[General]
window_manager=i3

5. 運命の起動テスト

⚠️ 注意！
実機画面が固まった場合に備えて、今繋いでいるこのSSHセッション（ターミナル）は絶対に閉じずに残しておいてください。

手動でログイン画面（SDDM）を立ち上げます。

sudo rc-service sddm start

🚨 万が一、画面が固まって操作できなくなった場合

残しておいた SSHの画面側 から以下のコマンドを打ち込んでください。SDDMを強制停止して元の黒い画面に戻れます。

sudo rc-service sddm stop

🎉 正常に動いた場合

  1. SDDM画面のセッション選択メニューから 「LXQt Desktop」 を選択します。
  2. hoge ユーザーでログインすれば、LXQtとi3が融合した快適なデスクトップが始まります！
