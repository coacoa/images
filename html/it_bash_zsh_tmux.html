bash / zsh / tmux の違い（メモ）

超要約
  • bash / zsh：コマンドを解釈・実行する シェル
  • tmux：画面分割・セッション維持をする 器

構造イメージ

[ ターミナル ]
   └─ tmux（画面管理・維持）
        └─ bash / zsh（コマンド解釈）
             └─ ls / vim / lf / git …

bash / zsh とは

  • シェル（コマンドを理解して実行）
  • 補完・パイプ・リダイレクト・環境変数を担当
ls | grep nix > files.txt

tmux とは

  • ターミナルマルチプレクサ
  • 画面分割、セッション保持、ssh切断後も作業継続
Ctrl-a |   # 横分割
Ctrl-a d   # デタッチ

bash と zsh の違い

項目bashzsh
標準多くのLinuxmacOS標準
補完控えめ強力
プラグイン少なめ多い
Vim派相性普通良い

よくある誤解

  • tmux はシェルではない（中で bash/zsh が動く）
  • 画面分割は tmux の機能

おすすめ運用

  • ログインシェル：zsh
  • スクリプト：bash
  • 画面管理：tmux
一言まとめ
tmux は「部屋」、bash / zsh は「中で話す言語」
