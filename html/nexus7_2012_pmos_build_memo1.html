STEP 1: pmaports の準備

grouperは最新pmaportsから削除されているため、削除直前のコミットを使う。

✅ pmaports clone & 正しいコミットへ checkout
git clone https://gitlab.postmarketos.org/postmarketOS/pmaports.git ~/pmaports
cd ~/pmaports
# grouperが削除されたコミット = f58f45db2d
# その一つ前をチェックアウト
git checkout f58f45db2d^
# → 23e6297d4a (linux-postmarketos-grate: upgrade to 6.6.0 stable)
✅ grouper の場所
device/testing/device-asus-grouper/
device/testing/firmware-asus-grouper/
⚠️ 注意: pmaportsはwork pathの外に置くこと

pmbootstrap の work path (~/.local/var/pmbootstrap) の中にpmaportsを置くとエラーになる。~/pmaports に置くこと。

STEP 2: NixOS での pmbootstrap 環境

STEP 4: gtk+3.0 ビルドエラーと対処

grouperはpulseaudioに依存→pulseaudioはgtk+3.0に依存。UIをconsoleにしても必要。

❌ エラー内容
../gtk/gtkprogresstracker.c:48:1: error: conflicting types for '_gtk_get_slowdown'
  have 'gdouble(gdouble)' vs 'gdouble(void)'

原因: 古いgtk+3.0のコードが新しいgcc (15.x) の厳しい型チェックに引っかかる。

🔧 修正方法: パッチを作成してAPKBUILDに追加

① パッチファイル作成

cat > ~/pmaports/temp/gtk+3.0/fix-gtk-get-slowdown.patch << 'EOF'
--- a/gtk/gtkprivate.h
+++ b/gtk/gtkprivate.h
@@ -96,7 +96,7 @@
-gdouble _gtk_get_slowdown ();
+gdouble _gtk_get_slowdown (gdouble factor);
EOF

② sha512sum 計算

sha512sum ~/pmaports/temp/gtk+3.0/fix-gtk-get-slowdown.patch

③ APKBUILD編集 (Pythonで確実に)

python3 << 'PYEOF'
with open('/home/hoge/pmaports/temp/gtk+3.0/APKBUILD', 'r') as f:
    content = f.read()

# sourceにパッチ追加 (タブ区切りに注意！)
content = content.replace(
    '\tgdk-wayland-Track-last-touch-serial-on-seat.patch\n\t"',
    '\tgdk-wayland-Track-last-touch-serial-on-seat.patch\n\tfix-gtk-get-slowdown.patch\n\t"'
)

# sha512sumsにハッシュ追加
content = content.replace(
    '4b8ecb3f...  gdk-wayland-Track-last-touch-serial-on-seat.patch',
    '4b8ecb3f...  gdk-wayland-Track-last-touch-serial-on-seat.patch\n<NEW_HASH>  fix-gtk-get-slowdown.patch'
)

with open('/home/hoge/pmaports/temp/gtk+3.0/APKBUILD', 'w') as f:
    f.write(content)
PYEOF

④ APKBUILDをリセットしたい場合

cd ~/pmaports
git checkout temp/gtk+3.0/APKBUILD
⚠️ ハマりポイント
  • APKBUILDのインデントはスペースではなくタブ。sedで編集する際は要注意。
  • sourceとsha512sumsの両方にパッチを追加しないとエラーになる。
  • パッチを変更したらsha512sumを再計算してAPKBUILDを更新する。
  • sed で複数回実行すると重複追加されるので git checkout でリセット推奨。

ADB 関連メモ

bootloader unlock後はUSBデバッグを再設定

unlock時にデータ全消去されるので、初期設定後に以下を再設定：

設定 → タブレット情報 → ビルド番号を7回タップ
設定 → 開発者オプション → USBデバッグ ON
ADB 接続確認
adb devices
