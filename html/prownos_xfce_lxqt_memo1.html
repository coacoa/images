PrawnOS XFCE → LXQt 移行 & x11vnc read 104 完全解決メモ

このメモは、PrawnOS（Debian Bullseye ベース）で デフォルト XFCE から LXQt へ完全移行し、 さらに Remmina の read 104 を確実に解決するための手順をまとめたもの。

■ 1. LXQt のインストール

sudo apt update
sudo apt install lxqt lxqt-core lxqt-session openbox sddm
ポイント：
  • LXQt の標準 WM は Openbox
  • SDDM を使うと LXQt と相性が良い

■ 2. SDDM を有効化（XFCE の LightDM を無効化）

sudo systemctl enable sddm
sudo systemctl disable lightdm
再起動後、ログイン画面で **LXQt Desktop** を選択。

■ 3. XFCE の WM（xfwm4）が残る問題の解決

LXQt を後付けすると、XFCE の WM が残って LXQt が「半起動」になる。 確認： 
ps aux | grep xfwm
動いていたら LXQt の設定を修正： 
nano ~/.config/lxqt/session.conf
以下を： 
window_manager=xfwm4
→ こう変更： 
window_manager=openbox
Openbox が無い場合： 
sudo apt install openbox
再起動。

■ 4. XFCE パネルが残る問題の解決

LXQt でも XFCE のパネルが残る場合がある。 XFCE パネルが動いていないか確認： 
ps aux | grep xfce4-panel
動いていれば停止： 
killall xfce4-panel
自動起動を削除： 
rm ~/.config/autostart/xfce4-panel.desktop
rm ~/.config/autostart/xfsettingsd.desktop
rm ~/.config/autostart/xfce4-power-manager.desktop
LXQt パネルを再起動： 
lxqt-panel --restart

■ 5. x11vnc の read 104 完全解決

read 104 の原因は **auth mismatch**。 SDDM は Xorg を -displayfd で起動するため、DISPLAY と auth が固定されない。 現在の Xauthority を確認： 
ls -l /var/run/sddm/
例： 
/var/run/sddm/{26a8aa67-3469-4c68-83ba-ad60f66ad68c}
これを x11vnc に明示的に渡す。

■ x11vnc.service を修正

sudo nano /etc/systemd/system/x11vnc.service
内容： 
[Unit]
Description=Start x11vnc at startup.
After=display-manager.service

[Service]
Type=simple
ExecStart=/usr/bin/x11vnc -auth /var/run/sddm/{26a8aa67-3469-4c68-83ba-ad60f66ad68c} \
 -noxdamage -usepw -rfbauth /etc/x11vnc.pass \
 -forever -loop -shared -repeat -display WAIT:0

[Install]
WantedBy=multi-user.target
反映： 
sudo systemctl daemon-reload
sudo systemctl restart x11vnc

■ 6. Remmina 側の設定

  • Security → VNC Password
  • 接続先は IPv4 を明示（例：vnc://192.168.x.x:5900）

■ 7. これで直る理由

  • -display WAIT:0 → SDDM の動的 DISPLAY を吸収
  • -auth /var/run/sddm/UUID → auth mismatch を完全排除
  • x11vnc が接続直後に落ちなくなる → read 104 が消える

■ 最終まとめ

PrawnOS の XFCE → LXQt 移行は、 SDDM の displayfd と XFCE の残骸のせいで複雑になりやすい。 しかし以下の 3 点を押さえれば確実に成功する：

  1. LXQt の WM を Openbox に固定
  2. XFCE のパネル・autostart を削除
  3. x11vnc に正しい Xauthority を指定し、DISPLAY は WAIT:0 を使う

これで LXQt + x11vnc + Remmina が完全に安定する。
