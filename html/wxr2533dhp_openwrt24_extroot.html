ExtRoot 設定メモ

USB 32GB を /overlay にマウントしてストレージ拡張する手順。SSH接続済みの状態から実施。

1
必要パッケージのインストール
opkg update
opkg install block-mount kmod-fs-ext4 e2fsprogs \
             kmod-usb-storage kmod-usb-storage-uas \
             parted usbutils
2
USB を挿してデバイス確認
fdisk -l
# または
ls /dev/sd*

通常 /dev/sda として認識される

3
パーティション作成・ext4 フォーマット
DISK="/dev/sda"
parted -s ${DISK} -- mklabel gpt mkpart extroot 2048s -2048s
DEVICE="${DISK}1"
mkfs.ext4 -L extroot ${DEVICE}
4
UUID 確認
block info ${DEVICE}

/dev/sda1 の UUID="xxxx-xxxx-..." をメモしておく

正常な出力例
/dev/sda1: UUID="e21f2b10-67b5-48d6-923c-..." VERSION="1.0" TYPE="ext4"
5
fstab 生成・設定
block detect > /etc/config/fstab
uci show fstab

@mount[0] に sda1 の UUID があることを確認してから実行：

uci set fstab.@mount[0].target='/overlay'
uci set fstab.@mount[0].enabled='1'
uci set fstab.@mount[0].enabled_fsck='1'
uci commit fstab
cat /etc/config/fstab — 正しい状態
config mount
        option target '/overlay'
        option uuid 'xxxx-xxxx-xxxx-xxxx'
        option enabled '1'
        option enabled_fsck '1'
6
overlay の内容を USB にコピー
mount /dev/sda1 /mnt
tar -C /overlay -cvf - . | tar -C /mnt -xf -
umount /mnt
reboot 前の確認（必須！）
mount /dev/sda1 /mnt
ls /mnt
# → upper  work  lost+found が表示されればOK
umount /mnt
注意
ls /mnt で upper と work が見えない場合は reboot 禁止。tar コピーをやり直すこと。
7
再起動・確認
reboot
# 再起動後
df -h
成功時の df -h 出力イメージ
/dev/sda1           29G   xx    xxG   x%   /overlay
overlayfs:/overlay  29G   xx    xxG   x%   /
成功
/ のサイズが約 30GB になっていれば ExtRoot 化完了！
!
起動しない場合の復旧手順
フォールバック仕様
USB を抜いてリセット → 内部 flash の overlay で起動する。文鎮化はしない。 
# USB なしで SSH 接続後、fstab を元に戻す：
uci set fstab.@mount[0].target='/mnt/sda1'
uci set fstab.@mount[0].enabled='0'
uci commit fstab
reboot
