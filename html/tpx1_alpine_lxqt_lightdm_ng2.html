🖥️ ThinkPad X1 Carbon 3rd Gen
Alpine 3.20 + DWM セットアップメモ

📌 背景・前提

項目内容
機種ThinkPad X1 Carbon 3rd Gen（Ivy Bridge / Intel HD 4000）
OSAlpine Linux 3.20
WMDWM 6.5（ソースビルド）
DMLightDM（autologin）
リモートx11vnc + Remmina
⚠️ Alpine 3.23 は使えない！
Mesa から i965_dri.so が削除されており、Ivy Bridge（HD 4000）は非対応。Xorg が起動不能になる。必ず Alpine 3.20 を使うこと。

1. 基本パッケージインストール

apk add xorg-server xf86-input-libinput xf86-input-evdev lightdm lightdm-openrc
apk add dwm dmenu st
apk add x11vnc
apk add j4-dmenu-desktop
apk add xdotool xinput

2. DWM ソースビルド

apk の DWM は config.h が存在しないためカスタマイズ不可。ソースからビルドする。

apk add gcc make musl-dev libx11-dev libxft-dev libxinerama-dev

cd ~
wget https://dl.suckless.org/dwm/dwm-6.5.tar.gz
tar xf dwm-6.5.tar.gz
cd dwm-6.5
cp config.def.h config.h

config.h の変更点

// MODKEYをWinキーに変更
#define MODKEY Mod4Mask

// dmenuをj4-dmenu-desktopに変更
static const char *dmenucmd[] = { "j4-dmenu-desktop", NULL };
make && sudo make install
✅ ビルド後は /usr/local/bin/dwm に配置される。apk のものを上書きするので apk del dwm 不要。

3. LightDM autologin 設定

# /etc/lightdm/lightdm.conf
[Seat:*]
autologin-user=youruser
autologin-session=dwm

DWM セッションファイル

# /usr/share/xsessions/dwm.desktop
[Desktop Entry]
Name=dwm
Exec=/usr/local/bin/dwm
Type=Application
⚠️ Exec=dwm だと古い /usr/bin/dwm が呼ばれる場合があるのでフルパス指定推奨。

4. x11vnc 設定

# 起動コマンド
x11vnc -display :0 -forever -passwd yourpassword -rfbport 5900

# .xinitrc に追記して自動起動
x11vnc -display :0 -forever -nopw -bg
exec dwm

Remmina 接続設定

項目値
プロトコルVNC
サーバーThinkPadのIP:5900
パスワード設定した場合のみ

5. sudo 設定

# root で実行
visudo
# 以下を追加
youruser ALL=(ALL) ALL

# または wheel グループに追加
adduser youruser wheel

6. USBキーボードが認識されない場合

✅ USBハブのポートを変えると認識されることがある。
xinput list で確認：DISPLAY=:0 xinput list 
# キーボードがXに認識されているか確認
DISPLAY=:0 xinput list

# Xorg ログ確認
grep -i input /var/log/Xorg.0.log

7. キーバインド一覧（DWM デフォルト + カスタム）

キー動作
Win + Pj4-dmenu-desktop 起動
Win + Enterターミナル（st）起動
Win + Shift + QDWM 終了（再起動）
Win + Shift + Cウィンドウを閉じる
Win + 1〜9タグ切り替え
