Web学習インタラクティブ教材 アイデアメモ

Flash/AS3時代の感覚を活かしつつ、HTML / CSS / JavaScript の学習と実用を兼ねた 「触れる教材」を作るためのメモ。

目的

  • HTML / CSS / JS を「動き」で理解する
  • ボタン操作・キー入力・音など即時フィードバック
  • 勉強用コンテンツそのものを自作する
  • Flash時代のインタラクティブ学習体験をWebで再構築

やりたいコンテンツ案

① HTML / CSS / JS 可視化教材

  • ボタンを押すと class が付く / 外れる
  • CSS が即座に反映される
  • 対応する CSS / JS のコードを同時表示
  • どの記述がどこに影響しているかを視覚的に理解

② キーボード練習（Vim 等）

  • 押したキーを画面に表示
  • hjkl 移動の可視化
  • operator + motion の流れを表示
  • 身体感覚で覚える練習用ミニアプリ

③ ミニゲーム / ミニアプリ

  • ボタンやキー入力で遊べる簡単ゲーム
  • 反応が楽しいことを重視
  • 学習の合間のモチベーション維持

④ ピアノ練習ミニアプリ

  • PCキーボードをピアノ鍵盤に見立てる
  • キー入力で音が鳴る
  • 視覚的に鍵盤が反応
  • Web Audio API の学習も兼ねる

共通コンセプト

  • 1 HTML = 1 テーマ
  • ローカルで完結（ビルド不要）
  • nvim / emacs どちらでも編集可能
  • GitHub Pages にそのまま載せられる

技術スタック（最小）

  • HTML：構造
  • CSS：見た目・状態の可視化
  • JavaScript：イベント・ロジック
  • SVG：図解・キー配置・構造説明

段階的ステップ案

  1. ボタンで CSS を切り替える HTML
  2. キーボード入力を表示する HTML
  3. キー入力 + 状態管理
  4. 音を出す（ピアノ）

メモ

  • Flash時代の timeline 的理解を JS に置き換える
  • CSS は「条件」、JS は「命令」
  • 教材を作ることで理解が定着する
