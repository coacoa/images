NixOS 26.05 移行記録：キーボードJP固定化の完全解決

作成日2026年6月1日
対象OSNixOS 26.05 (Warbler)
環境設定OS言語: 英語 (en_US.UTF-8) / キーボード: 日本語配列 (JIS)
利用環境LXQt / DM: SDDM / 入力制御: Kanata (v1.11.0) + IBus

1. トラブルの本質（なぜ勝手にUS配列に戻るのか）

NixOS 25.05環境までは正常に日本語配列が維持されていたが、26.05移行（およびLXQt/IBus環境）において、ログイン直後にキーマップがUS配列に強制フォールバックする怪奇現象が発生。

競合のメカニズム：
  1. i18n.defaultLocale = "en_US.UTF-8"; の影響で、OS起動時に内部的にUS配列へのフォールバック圧力がかかる。
  2. 低レイヤーキーリマップツール Kanata が起動し、物理キーを横取りして仮想キーボードを生成する。この際、Kanata側の (defsrc) がUS配列ベースの定義であるため、新OSのデバイス管理機構が「これはUSキーボードである」と厳密に解釈してしまう。
  3. デスクトップ（LXQt）やIBusの起動タイミングと重なり、システム側の services.xserver.xkb.layout = "jp"; や通常の setxkbmap jp が上書きされ無効化されていた。

2. 回避された「カオス化」ルート（不採用案）

  • Kanata設定のJIS化案： Kanataの defsrc 自体を minus や at などのJISキーコードに書き換えるアプローチ。Kanataパーサーのバージョンによる互換性（Unknown keyエラー）や、既存の作り込まれた複雑なレイヤーマクロ（ナビゲーション・精密・ドラッグモード等）の挙動崩壊のリスクが高いため却下。
  • Home Managerの home.keyboard 制御案： 裏で独自の初期化ファイルを生成するため、システム側やデスクトップ側の設定と真っ向から衝突しリセットを誘発するため無効化。

3. 最終解決策：タイミングをずらした遅延実行 (Sleep 3)

もっとも安全かつ確実な手法として、「KanataがUS配列として起動しきった『直後』を狙い、Home Managerの自動起動から独立したレイヤーで setxkbmap jp をガツンと叩き込む」という遅延ハックを採用。

ステップ 1：共通キーボード設定のクリーンアップ

Home Manager側で予期せぬキーボードリセットを発生させていた home.keyboard をコメントアウトし、GNOME等の特定環境のみdconfを安全に叩く構成に制限。

shared-keyboard-config.nix (該当箇所)
############################################################
# Keyboard (XKB)
############################################################
# ⚠ LXQt等の環境でUSフォールバックを引き起こすためコメントアウト
# home.keyboard = {
#   layout = "jp";
#   options = [ "ctrl:nocaps" ];
# };

# システムがGNOME有効時のみ、安全にdconf経由でCtrl:nocapsを適用
dconf.settings = lib.mkIf config.services.xserver.desktopManager.gnome.enable {
  "org/gnome/desktop/input-sources" = {
    xkb-options = [ "ctrl:nocaps" ];
  };
};

ステップ 2：独立した自動起動モジュールの作成

LXQt起動完了後、3秒待機してから日本語配列を上書き強制するデスクトップファイルを生成する専用モジュールを新設。

home.lxqt.setxkbmap.jp.nix (新規作成)
{ pkgs, ... }:

{
  # LXQt/X11のログイン時に、Kanataが立ち上がるのを少し待ってから jp 配列を強制する
  xdg.configFile."autostart/setxkbmap.desktop".text = ''
    [Desktop Entry]
    Type=Application
    Version=1.0
    Name=SetXKBMapJP
    Comment=Force Japanese Keyboard Layout after Kanata starts
    Exec=sh -c "${pkgs.coreutils}/bin/sleep 3 && ${pkgs.xorg.setxkbmap}/bin/setxkbmap jp"
    Terminal=false
    StartupNotify=false
    X-LXQt-NeedDE=true
  '';
}

ステップ 3：Home Managerへのインポートと反映

home.endeavor.nix 等の imports = [ ... ]; に上記 ./home.lxqt.setxkbmap.jp.nix を追記し、通常通りリビルドを実行。

sudo nixos-rebuild switch

4. 構築完了後の効果

完全勝利：
  • OS表記、システムロケールは綺麗な英語（en_US）のまま維持。
  • ログインした直後から自動で @ や _ などの記号位置が1文字もズレない快適な日本語配列（JIS）に固定。
  • 苦労して作成したKanataのレイヤー機能（速度切り替え、マウス移動など）は1行も変更せず完全動作。
  • 設定を別ファイルに疎結合化したため、今後のPC増設時にも1行インポートするだけで使い回し可能。
