OpenWrt 23 + AdGuard Home 最適構成メモ

■ 構成の目的

OpenWrt 23 上で AdGuard Home を DNS サーバーとして動かし、
LAN 端末の広告ブロック・フィルタリングを高速かつ安定して行う構成。
dnsmasq は DHCP のみ担当し、DNS は完全に AdGuard Home に任せる。

■ 全体構成

  • DNS: AdGuard Home（ポート 53）
  • DHCP: dnsmasq（OpenWrt 標準）
  • NAT: OpenWrt（nftables）
  • LAN: 192.168.240.0/24
  • LAN Gateway: 192.168.240.1（OpenWrt）

■ dnsmasq の設定（DHCP のみ）

dnsmasq の DNS を無効化し、DHCP のみ動作させる。

config dnsmasq
    option port '0'   ← DNS を無効化

config dhcp 'lan'
    list dhcp_option '6,192.168.240.1'  ← DNS = AGH

■ AdGuard Home の設定

設定ファイル: /etc/adguardhome.yaml

● DNS ポートを 53 に変更

dns:
  port: 53

● sed で変更する場合

sed -i 's/port: 5353/port: 53/' /etc/adguardhome.yaml
killall AdGuardHome
/usr/bin/AdGuardHome -c /etc/adguardhome.yaml -w /var/adguardhome &

■ NAT（nftables）

OpenWrt 23 は nftables を使用。MASQUERADE が有効なら OK。

nft list ruleset | grep masquerade

■ LAN 端末の動作

LAN 端末は DHCP により以下を受け取る：

  • IP: 192.168.240.x
  • Gateway: 192.168.240.1
  • DNS: 192.168.240.1（AGH）

■ なぜ AGH は 53 番ポートで動かす必要があるのか？

LAN 端末は DNS を 必ず 53 番ポートで問い合わせる。
DHCP は DNS の「ポート番号」を配布できないため、
AGH を 5353 のままにすると LAN 端末は DNS に接続できない。
そのため AGH を 53 にするのが最適構成。

■ この構成のメリット

  • LAN 端末は自動で AGH を DNS として利用
  • dnsmasq は DHCP のみ → シンプルで安定
  • AGH が DNS を完全に担当 → 高速・高機能
  • OpenWrt の NAT はそのまま利用 → 高速
  • 再起動しても設定は永続保存

■ 推奨追加設定

  • AGH の DoH/DoT アップストリーム設定
  • フィルタリストの最適化
  • ログ保存先を外部ストレージに変更
  • OpenWrt の DNSSEC 無効化（AGH が担当するため）
