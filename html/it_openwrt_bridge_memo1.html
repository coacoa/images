2033GR ブリッジ + AdGuard Home 設定備忘録

このメモは、親機(F660T)配下で2033GRをブリッジ（APモード）として動作させ、広告ブロック(AGH)を有効にした際の設定ログです。

1. ネットワーク基本情報

項目設定値
2033GR IPアドレス192.168.1.250
デフォルトゲートウェイ192.168.1.1 (F660T)
AGH 待ち受けポート5353
AGH 管理画面http://192.168.1.250:3000

2. OpenWrt 内部パスの設定 (SSH)

dnsmasq(53番)を窓口にし、処理をAGH(5353番)へ投げる設定です。

# dnsmasqの上流をAGHに固定
uci del dhcp.@dnsmasq[0].server
uci add_list dhcp.@dnsmasq[0].server='127.0.0.1#5353'
uci set dhcp.@dnsmasq[0].noresolv='1'

# IPv6のDNS広告を停止（広告回避防止）
uci set dhcp.lan.ra_dns='0'
uci set dhcp.@dnsmasq[0].filter_aaaa='1'

uci commit dhcp
/etc/init.d/dnsmasq restart
/etc/init.d/network restart

3. AdGuard Home 側の設定

  • DNS設定: 待ち受けポートを 5353 に変更。
  • 上流DNSサーバー: 8.8.8.8 や 1.1.1.1 を指定（自分自身に向けない）。
  • ブロックモード: Default (IPアドレス 0.0.0.0 を返す) に設定。

4. Androidスマホ側の設定（最重要）

Androidの「DNS 2」に Google DNS(8.8.8.8)等が入っていると広告が出てしまうため、必ず削除します。

  • IP設定: 静的 (Static)
  • DNS 1: 192.168.1.250
  • DNS 2: 空欄、または 0.0.0.0 / 127.0.0.1
  • プライベートDNS: オフ

5. 動作確認コマンド

# 内部で正しくAGHが呼ばれているかテスト
nslookup google.com 127.0.0.1
