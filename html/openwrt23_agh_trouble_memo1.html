OpenWrt + AGH DNS復旧マニュアル

トラブル状況： ネットが繋がらない、またはAndroidで「接続なし」になる。
原因： Dnsmasq(53)とAGH(5353)の連携ミス、またはDNSループ。

1. 疎通確認（原因の切り分け）

SSHで入り、以下のコマンドを実行する。

# 外の世界(IP)と繋がっているか
ping -c 3 8.8.8.8

# AGH(5353)が名前解決できているか
nslookup google.com 127.0.0.1:5353

※ AGH経由でIPが返ってこない場合は、AGH管理画面(:8080)の「アップストリームDNS」に 8.8.8.8 を設定する。

2. DNS設定の完全修正（一本化）

Dnsmasqの設定をAGH専用の転送役に固定する。

# 転送先をAGHのみにする
uci del dhcp.@dnsmasq[0].server
uci add_list dhcp.@dnsmasq[0].server='127.0.0.1#5353'

# WAN側DNSを無視（ループ防止）
uci set dhcp.@dnsmasq[0].noresolv='1'

# 反映
uci commit dhcp
/etc/init.d/dnsmasq restart

3. Androidの「接続なし」対策

数日で不安定になる場合は、Android側の設定を確認：
  • プライベートDNS: 「オフ」にする（自動だとポート853を使おうとして詰まる）。
  • IPv6 DNS: 繋がらない場合は、OpenWrt側でIPv6を無効化するか、IPv6 DNSもAGHに向ける。

構成図（正常時）

  • 端末 → [Port 53] → Dnsmasq
  • Dnsmasq → [Port 5353] → AdGuard Home
  • AdGuard Home → [外部] → Internet

※ これで直らない場合は、一度ルーターの再起動を行い、netstat -tulpn | grep 53 でポートの競合がないか再確認してください。
