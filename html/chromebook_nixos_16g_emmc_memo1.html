NixOS 容量不足（16GB eMMC）対策メモ

Chromebook等の小容量ストレージ環境で、SDカードを /nix ストレージとして活用し、ビルドエラーを回避した全記録。

最終結果： 本体(eMMC) 16GB のうち 7.5GB 使用で余裕を確保。実体は SDカード 30GB 側で運用。

1. 限界突破のための掃除（延命処置）

まずはビルドとコピーができるだけの隙間を作る。

sudo nix-collect-garbage -d
sudo journalctl --vacuum-time=1s
sudo nix-store --optimize

2. SDカードの準備

パーティションを分けず、sda まるごとを ext4 でフォーマットし、容量を最大化する。

sudo mkfs.ext4 -F -L nix_storage /dev/sda
sudo mount /dev/sda /mnt

3. データの引っ越し（心臓部のコピー）

本体の /nix を SDカードへ。属性を保持したまま同期する。

sudo rsync -aAXv /nix/ /mnt/

4. 設定の恒久化 (configuration.nix)

ここが最大のポイント。SDカードをシステムの /nix として定義する。

fileSystems."/nix" = {
  device = "/dev/disk/by-label/nix_storage";
  fsType = "ext4";
  options = [ "noatime" "nodiratime" "discard" ];
  neededForBoot = true;
};

5. トラブルシューティング：ビルド不整合の解消

マウント切り替え後の Connection refused や No such file エラーへの対処。

  1. 手動で SDカードを /nix に被せる：sudo mount /dev/sda /nix
  2. デーモン再起動：sudo systemctl restart nix-daemon
  3. 強制ビルド：sudo nixos-rebuild switch --repair

6. 勝利の確認

再起動後、以下のコマンドで 30GB 認識されていれば完了！

df -h /nix
注意： 今後、SDカードを抜いた状態で起動するとシステムが壊れるため、常に挿入しておくこと。
