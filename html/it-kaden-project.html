家電化APプロジェクト（Kaden Project）メモ

コンセプト
古いノートPCを再利用し、電源ON/OFFとLAN接続だけで動作するAP型コミュニケーション装置を作る。
PCとして使わせず、家電・業務機器のように扱えることを最優先とする。
  • エコ／リサイクル
  • ローカル完結（通信内容は外部サーバに出ない）
  • 高齢者・子供・ペット・キャンプ用途

基本思想（最重要）

  • 操作は原則 電源ON / OFF のみ
  • LAN接続は挿すだけ、設定不要
  • キーボード・マウスはデバッグ時のみ使用
  • 画面は操作用ではなく「状態表示専用」

OS / 起動方式

  • NixOS 25（legacyOS想定）
  • USB 32GB に完全同一イメージを作成
  • USB = ファームウェア（コピーして何台でも使用）
  • read-only 運用を前提

ネットワーク設計

Wi-Fi

  • 全ノードがAPとして動作
  • 同一SSID・同一パスワード
  • 利用者からは「1つのネットワーク」に見える

LAN

  • 有線LANはノード間連携専用
  • LAN接続で自動クラスタ検出
  • LANを抜いても単独APとして継続

ノード識別

  • AP名（SSID）では識別しない
  • MAC + USB UUID からノードID生成
  • 例：ap-node-1f3a

起動フロー

1. 電源ON
2. USBからNixOS起動
3. systemd services 起動
4. ネットワーク自動判定
5. Wi-Fi AP 起動
6. LANがあればクラスタ探索
7. 状態を画面に常時表示

デバッグ・表示設計

  • 自動ログイン
  • tmux 常駐
  • ターミナルは状態表示専用

想定用途

  • 高齢者世帯の会話・見守り
  • 子供・ペット見守り
  • キャンプ・災害時ローカル通信
  • ドアホン代替

プロジェクト方針

  • 機能を削ることを価値にする
  • 設定不要・説明不要
  • まずは2台LAN接続で自動認識まで
