いま何が起きているか

端末（例: 192.168.20.50）から出たパケットは、

  • NixOS ルーター（192.168.20.1）で NAT されて 送信元が 10.24.219.47 に書き換わる
  • スマホは 10.24.219.47 を自分の「内側のアドレス」として扱い、さらにキャリアのグローバルIPに変換してインターネットへ出す

つまり流れは：
192.168.20.x → (NixOS NAT) → 10.24.219.47 → (スマホ NAT) → グローバルIP
という「二重 NAT」になっている。

なぜ家庭の LAN は 192.168.x.x で、テザリングは 10.x.x.x なのか

インターネットの仕様（RFC1918）で、ローカル専用の「プライベートIPアドレス」が3つの範囲として予約されている：

  • 10.0.0.0 ～ 10.255.255.255（10.x.x.x）
  • 172.16.0.0 ～ 172.31.255.255（172.16.x.x）
  • 192.168.0.0 ～ 192.168.255.255（192.168.x.x）

これらはどれも「インターネットに直接出ない番号帯」で、
ルーターやスマホがそれぞれ自分の「ローカルの島」を作るために使っている。

  • 家庭用ルーターは扱いやすいので 192.168.x.x をよく使う（例: 192.168.0.1, 192.168.1.1）
  • スマホのテザリングは広い範囲が欲しいので 10.x.x.x をよく使う

だから、あなたの環境で：

  • NixOS の AP 側 LAN が 192.168.20.x
  • スマホの USB テザリング側が 10.24.219.47

という構成になっていてもまったく問題なくて、
それぞれ別の「ローカルの島」として動き、その間を NAT が橋渡ししている。
