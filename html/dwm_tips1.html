dwm / sway / i3 / Hyprland 比較

WM方式長所短所
dwm X11（極小・Cソース編集）
  • 最軽量・最速
  • キーバインドがすべて C で管理 → 変更が永久
  • ソースを読んで理解する楽しさ
  • パッチ文化で自由度最高
  • 設定変更に再コンパイルが必要
  • Wayland の時代には古い（X11限定）
  • マルチモニタや複雑なコンポジットは弱い
sway Wayland の i3 互換 WM
  • i3 と同じ設定で Wayland が使える
  • 超安定・クセが少ない
  • dwl よりドキュメントが豊富
  • アニメーション弱い
  • カスタム性は Hyprland に劣る
i3 X11 タイリング WM
  • dwm より習得が簡単
  • 設定ファイルだけで完結
  • ドキュメントや例が豊富
  • 軽いが dwm ほどではない
  • Wayland 移行が前提なら将来は sway へ
Hyprland Wayland：アニメーション系の新世代 WM
  • 見た目が最強（アニメーション・ぼかし）
  • 設定が読みやすい
  • 複雑なウィンドウ操作・レイアウトが簡単
  • 軽さは dwm より劣る
  • 比較的新しく安定性が環境差で変動

dwm を軸にするとどうなる？

  • 最軽量の体験をまず知る（基準値になる）
  • キーバインド・レイアウトの思想が理解しやすい
  • sway/i3/Hyprland が「どれくらい重いのか」が感覚で分かる
  • 自分が「どこまで自動化したいか」を考えられる

dwm の基本操作（最初に覚えるところ）

操作キー
端末を開くMod + Shift + Enter
アプリを閉じるMod + Shift + C
次のウィンドウへ移動Mod + J
前のウィンドウへ移動Mod + K
現在のウィンドウを master に変更Mod + Enter
レイアウト変更（tile / monocle / floating）Mod + Space
フロート切替Mod + Shift + Space
ウィンドウを閉じるMod + Shift + C
再起動Mod + Shift + R（パッチ必要）

どれが「dwm の思想に近い」か？

  • dwm → 本家
  • dwl → dwm の Wayland 版（まだ発展途上）
  • i3 / sway → ルールベースは近いが「設定ファイル方式」
  • Hyprland → 完全に別物（アニメ重視）
